## Context Setting
Establish what was already done before this A/B test. What are some questions you can ask to get more information on what has already been measured, analyzed, or attempted? 
* Did we previously measure how **increasing** the number of words/slide impacts viewing time? This can help us determine a words/slide sweet-spot.
* During what time frame was the data collected (i.e. 3 days, 1 week, 1 month, etc.)? We need to determine if this was a sufficient period of time considering temporal patterns of usage. 
* Have we previously analyzed the effect of viewing time on ad revenue (if this is a stakeholder goal - see below)?


## Goal Alignment 
Align with stakeholders on what success means before interpreting any numbers.
* Is increased viewing time a success? If so, our goal is increased engagement (perhaps for ad revenue or user data collection).
* Is our goal to increase user Test Scores or Course Completion Rates? If so, then we want to improve measures of learning comprehension.
* How would we rank these goals in order of importance (Test Scores vs. Course Completion Rates vs. Engagement?)?


## Define Over-performance 
Consider what it would mean if the treatment dramatically exceeded expectations.
* By reducing the number of words/slide, we risk oversimplifying the educational content. In that case, higher test scores and course completion rates can suggest that the course has become too easy.
* By changing the number words/slide we should be mindful of unreasonably high or low viewing times. This can suggest user difficulty with learning comprehension, or skimming because the content is too simple or too dense. 
* So far, decreasing the word count has increased the avg. viewing time, which suggets that users are more engaged with the educational content. We need to understand what is the words/slide sweet-spot wherein engagement (or viewing time) is reasonable: not too high suggesting educational strain, but not too low which can suggest skimming for brevity.
